# Mes Scripts TCL (exhausif) :

* [TCL-Youtube-Link](https://github.com/ZarTek-Creole/TCL-Youtube-Link)

Ce script récupère des informations sur les titres Youtube à l'aide de l'API Youtube V3.
Permet de rechercher des titres avec des mots clefs
Il écoute les liens youtube sur les salons et affiche les informations des titres
Les annonces sont totalement personalisable : Couleurs, informations : Durée, etc


* [TCL-DictionaryAPI](https://github.com/ZarTek-Creole/TCL-DictionaryAPI)

Ce script récupère des informations sur les mots à l'aide de l'[API Dictionary v2](https://dictionaryapi.dev), il support les langues suivantes: en,hi,es,fr,ja,ru,de,it,ko,pt-br,ar,tr

* [TCL-SCRIPT-MANAGER Aka TSM](https://github.com/ZarTek-Creole/TCL-SCRIPT-MANAGER)

TSM (TCL-SCRIPT-MANAGER) est unTSM gestionnaire de scripts TCL en terminal qui vous permet d'installer, de mettre à jour et de désinstaller facilement des scripts TCL. Il est basé sur un système de "sources listes" contenant des catalogues de scripts (sur git) personnalisables, comme un genre de APT pour script TCL Eggdrop.


* [TCL-ReplicaServ-Service](https://github.com/ZarTek-Creole/TCL-ReplicaServ-Service)

Version eggdrop de l'ancien script "Les poupées linkeuses". Permet de répliqué sur votre IRCD les users/messages d'un ircd/salon distance


* [TCL-PROXYCHECK](https://github.com/ZarTek-Creole/TCL-PROXYCHECK)

Ce script "VPN CATCHER" verifie les informations de connexions des utilisateurs qui ce connecte a votre IRCD et permet de bannir les connexions les moins sécuriser. 


* [TCL-MOCKS](https://github.com/ZarTek-Creole/TCL-MOCKS)

Ce script manage le logiciel de proxy ['My Own soCKs Server' aka mocks](https://sourceforge.net/projects/mocks)


* [TCL-EvaServ](https://github.com/ZarTek-Creole/TCL-EvaServ)

EvaServ est un ensemble de Services IRC écris en TCL et fonctionnant avec un eggdrop (v1.9+) pour la gestion de votre réseau IRC. Il a été testé sur un Unrealircd (v5.0+) qui utilise les nouveaux protocoles IRC, il est compatible avec la nouvelle génération de serveurs d'IRCD.


* [TCL-UJL Aka Unrealircd Json Logs](https://github.com/ZarTek-Creole/TCL-UJL)

Outils de gestions du cap unrealircd json logging


* [TCL-SHOWINFO](https://github.com/ZarTek-Creole/TCL-SHOWINFO)

Scripts qui permet d'afficher du contenue par titre


* [TCL_Exec](https://github.com/ZarTek-Creole/TCL_Exec)

TCL_Exec permet d'executer des commandes TCL via votre salon IRC par votre robot.

* [TCL-DebugLoade](https://github.com/ZarTek-Creole/TCL-DebugLoade)

Ce script ce charge en premier dans la config eggdrop. DebugLoader permet de charger des scripts TCL. Il redirige les erreurs dans un fichier ${::botnet-nick}_Error.log, ainsi que sur un channel 'Channel_Debug'. Très pratique pour les developpers, afin de suivre les erreurs, mais également il permet de ne pas "planté" le robot en cas de probleme…


* [TCL-AuthIRC](https://github.com/ZarTek-Creole/TCL-AuthIRC)

Ce script ce charge en premier dans la config eggdrop. AuthIRC permet d'identifier votre robot aux services ANOPE. Il s'auto-invite & join les channels desiré.



* [TCL-ZCT Aka ZarTek-Creole Toolkit](https://github.com/ZarTek-Creole/TCL-ZCT)

ZCT est un ensemble d'outils TCL conçu pour fonctionner avec les TCL de ZarTeK-Creole Tools

* [fix.eggdrop.cmd.tcl](https://github.com/ZarTek-Creole/tcl-fix.eggdrop.cmd/blob/main/fix.eggdrop.cmd.tcl)

Fixed some eggdrops commands in logic.

* [tcl-echo-scripts](https://github.com/ZarTek-Creole/tcl-echo-scripts)

Permet de creer un systeme de prebot -alimenter par des salons de ADDPRE- via un botnet Fonctionne selon le principe de MASTER-SALVE. C'est à dire que les SLAVES sont connecte via le botnet a un MASTER. Le MASTER recois les informations des slaves pour ajouter en base de donneées. Chaque SLAVES ont un script ESCRIPTS-FCT_BOT:ECHOs.tcl qui ecoute les salons pour envoyer les annonces !ADDPRE etc au MASTER. Pour interroger la base de donneés, le script Echo-CMDS.tcl permet de repondre aux commandes

* [TCL-BYPASS-WHO-BLOCKED](https://github.com/ZarTek-Creole/TCL-BYPASS-WHO-BLOCKED)

Certain IRCD empêche les Eggdrops en modifiant ou bloquant les réponses WHO/WHOX. ce script permet de faire fonctionner eggdrop tout de même.
